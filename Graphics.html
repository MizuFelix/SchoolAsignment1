Grafiken


Zurück

Grafiken sind die Formen und Körper die man während des spielens

im Spiel sieht. Dazu muss der Code mit der Grafikkarte kommunizieren.


Wenn man ein externes Werkzeug für die Entwicklung

benutzt, sollten Grafiken direkt davon "erstellt" werden.


Wenn man allerdings sein Spiel komplett selbst schreibt, brauch man

einen Weg seine Grafiken zu schreiben. Das ist wo Graphics APIs

ins Spiel (;P) kommen.


API = Application Programming Interface


Sie sprechen mit der Grafikkarte und sagen ihr wann sie was auf den

Bildschirm zeichnen muss.

Diese kann man in sein Programm einbinden. Die Grafik API freundlichste

Programmiersprache ist C++. Das heißt für diese Programmiersprache gibt

es die meisten dafür entwickelten APIs.


Beispiele:

  • Khronos Group - OpenGL (Verfügbar auf iOS, Android, Mac, Linux und Windows)
  • Microsoft - DirectX (Verfügbar auf Windows und XBOX)
  • Apple - Metal (Verfügbar auf iOS, iPadOS und Mac)

Die Sache ist: Grafikprogrammierung ist meist schwer und aufwendig,

deswegen wird empfohlen mit OpenGL (Open Graphics Library),

einer eher einfachen API, anzufangen.
